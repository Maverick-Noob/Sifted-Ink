## 1. 快速开始

### 1.1 安装

```bash
# 克隆项目
git clone <repo-url>
cd novel_preactor

# 安装依赖
pip install -r requirements.txt
```

### 1.2 设置 API Key

```bash
# Linux / macOS
export SIFTED_INK_API_KEY="sk-ant-..."

# Windows PowerShell
$env:SIFTED_INK_API_KEY="sk-ant-..."

# 或使用各提供商的专属环境变量
export ANTHROPIC_API_KEY="sk-ant-..."    # Anthropic Claude
export DEEPSEEK_API_KEY="sk-..."         # DeepSeek
export OPENAI_API_KEY="sk-..."           # OpenAI
export MOONSHOT_API_KEY="sk-..."         # Moonshot / Kimi
# ... 更多见 README
```

### 1.3 启动

```bash
# 推荐：Web 界面
python -m novel_preactor --web
# 浏览器访问 http://localhost:8000

# 或命令行模式
python -m novel_preactor --config story_config.yaml
python -m novel_preactor --interactive
```

## 2. Web 界面详解

启动后访问 `http://localhost:8000`，整个页面分为多个卡片区域：

### 2.1 主角设定

| 字段 | 说明 | 必填 |
|------|------|:---:|
| 姓名 | 主角姓名 | ✅ |
| 性格、动机、能力 | 自由描述，越详细 AI 越能把握角色 | ✅ |
| 团队角色 | 如「队长」「智囊」，多人团时有用 | |
| 能力/技能 | 角色的特殊能力 | |
| 登场章 | 在指定章节登场，0=AI 自行决定 | |

**多人主角团**：点击「➕ 添加主角」按钮，每增加一行就多一个主角。主角人数会自动更新。删除行即可减少。

### 2.2 世界背景

描述故事的时空设定：时代（中世纪/现代/未来）、魔法/科技水平、种族、势力等。越具体，AI 构建的世界越丰富。

### 2.3 故事框架

| 字段 | 说明 |
|------|------|
| 故事开头 | 初始场景或事件，200-500 字效果最佳 |
| 故事结尾 | 期望的结局。**可留空**，AI 会自动设计一个有戏剧张力的结局 |

### 2.4 生成参数

| 参数 | 默认 | 说明 |
|------|------|------|
| 目标字数 | 8000 | 最终小说的大致长度。AI 会按剩余字数动态调整每章篇幅 |
| 并行版本 | 3 | 同时探索的故事版本数。越多越可能找到精彩情节，但消耗更多 Token |
| 章节上限 | 30 | 单版本最多允许的章节数，防止无限循环 |
| NPC 上限 | 30 | 故事中最多同时存在的 NPC 数量 |
| Token 预算 | 0（不限） | 设置上限可控制成本 |
| 质量模式 | 平衡 | 平衡=速度优先 / 质量优先=更多 NPC 交互+更长超时 |

### 2.5 主角团模式

| 模式 | 适用场景 | 每章行为 |
|------|---------|---------|
| **聚光灯轮换** | 群像剧（如海贼王） | 每章聚焦 2-3 人，其余作为背景，章间轮换 |
| **团队 Agent** | 团队冒险（如复仇者联盟） | 一次调用覆盖全员，侧重团队互动 |
| **全并行** | 需要每人独立内心独白 | 每个主角独立调用 Agent，质量最高但最慢 |

### 2.6 NPC 处理策略

| 策略 | 速度 | 适用 |
|------|:---:|------|
| 异步并行批处理 | 中等 | NPC 有独立对话和行动，同时发出请求 |
| 场景过滤 | 快 | 只处理当前场景直接涉及的 NPC |
| 叙事代劳 | 最快 | NPC 不独立调用，由叙事 Agent 一笔带过 |

### 2.7 指定 NPC

点击「➕ 添加 NPC」可以预定义角色：

| 字段 | 说明 |
|------|------|
| 姓名 | NPC 的名字 |
| 关系 | 与主角的关系：朋友/敌人/导师/恋人/对手/中立 |
| 重要性 | 主线关键/支线重要/路人 |
| 背景故事 | AI 会在合适时机融入剧情 |
| 登场/退场章 | 指定在哪些章节出现或离开（0=自动） |

AI 仍可自由创建额外 NPC，用户定义的角色会被优先使用。

### 2.8 写作风格

- **搜索框**：输入作家名或关键词筛选 183 位作家
- **公版安全** ✅：已故 50+ 年的作家，可安全使用
- **在世作家** 🟡：选择后会显示法律提示
- **自定义风格**：展开「添加自定义风格」自行定义

不选则使用默认风格。

### 2.9 前附文

可选在小说正文前添加：
- **目录**：自动从章节标题生成
- **引言**：书名+标语+主角名
- **楔子**：AI 生成 200-400 字神秘开篇
- **人物表**：主角和主要地点角色列表

### 2.10 API 设置

1. 选择**模型供应商**（13 家可选）
2. 模型名自动填入（可修改）
3. API Key 可留空——自动从环境变量读取

## 4. 核心概念

### 4.1 预演机制

选墨集不是直接生成小说，而是先「预演」多条故事线：

```
用户输入 → 自动结局（如未指定）
         → 生成 3 份不同角度的大纲 → 评估 → 选最优
         → 3-5 个版本并行推演（各探索不同分支）
         → 四维度评估 → 选最优版本
         → 起名 + 生成前附文 → 输出文件
```

### 4.2 Agent 角色

| Agent | 职责 |
|-------|------|
| 叙事导演 | 每章决策：推进方向、引入/退场 NPC、把握节奏 |
| 主角 Agent | 根据角色设定做出行动、对话、决策 |
| NPC Agent | 根据自身设定和场景做出反应 |
| 叙事 Writer | 将各方行动整合为优美叙事文字 |
| 评估师 | 四维度打分选出最佳版本 |

### 4.3 四维评估

| 维度 | 说明 |
|------|------|
| 戏剧张力 | 冲突是否激烈、情节是否吸引人 |
| 角色成长 | 主角是否有清晰的变化弧线 |
| 逻辑一致性 | 情节是否自洽、行为是否符合设定 |
| 结局匹配度 | 实际结局与目标结局的接近程度 |

## 5. 功能详解

### 5.1 自动结局

如果故事结尾留空，AI 会在预演开始前设计一个有戏剧张力的结局。支持的结局类型：反转胜利、悲壮牺牲、温暖和解、开放式、宿命轮回等。

### 5.2 节奏控制

系统自动实施三幕剧结构：
- **前 30%**：自由展开，埋下伏笔
- **30-50%**：引入核心矛盾
- **50-75%**：全力推进主线
- **75%+**：收束所有线索，禁止新支线

连续 3 章无明显进展时，系统会强制引入变化。

### 5.3 每章字数控制

AI 会根据「目标总字数 ÷ 剩余章节数」动态计算每章目标字数。写多了会自动收紧，写少了会适当放宽，确保最终接近目标。

### 5.4 NPC 名字风格

为 NPC 选择命名风格：中文/日文/西方/俄式/混合。Director 会从对应的名字库中选择或参考命名。

### 5.5 书名命名

支持 7 种风格：直白陈述、诗意意境、悬念钩子、人物标签、物件意象、时间地点锚定、对话金句。章节标题也会采用相同的命名风格。

### 5.6 多格式导出

| 格式 | 特点 | 依赖 |
|------|------|------|
| Markdown | 原生格式 | — |
| TXT | 纯文本，章节装饰分隔 | — |
| EPUB | 电子书标准，CSS 排版 | `ebooklib` |
| PDF | A5 书页尺寸，中文排版 | `fpdf2` |
| MOBI/AZW3 | Kindle 设备 | Calibre |

## 6. 输出文件说明

生成完成后，`output/{run_id}/` 目录下包含：

| 文件 | 内容 |
|------|------|
| `{书名}_{时间戳}_v{版本}.md` | 完整小说（Markdown） |
| `eval_log.json` | 评估日志（各版本评分、NPC 记录、Token 消耗） |
| `{书名}_{时间戳}_v{版本}_params.yaml` | 生成参数（可复用） |

## 7. 常见问题

### Q: 所有版本都超时怎么办？

- 减少并行版本数（设为 1 或 2）
- 减少章节上限（如 15-20 章）
- 选择「叙事代劳」NPC 策略
- 选择「质量优先」模式（自动延长超时）

### Q: 始终无法到达结局？

- 确保章节上限不要太低（建议 ≥ 20）
- 检查故事结尾描述是否清晰可达
- 降低目标字数与章节上限的比例

### Q: 生成速度太慢？

- 减少主角团人数
- NPC 策略选择「叙事代劳」
- 主角团模式选择「聚光灯」
- 降低并行版本数

### Q: API Key 报错？

- 确认环境变量名正确：`SIFTED_INK_API_KEY` 或各提供商的专属变量
- 确认 API Key 有效且余额充足
- Web UI 中留空 API Key 也会自动读取环境变量

### Q: 想复现之前的生成？

使用输出目录中的 `_params.yaml` 文件：
```bash
python -m novel_preactor --config output/{run_id}/*_params.yaml
```

## 8. 配置参考

### 完整 YAML 示例

```yaml
# 故事设定
protagonist_name: "林远"
protagonist_traits: "勇敢但冲动，20岁。擅长剑术，初级火系魔法。"
world_setting: "架空奇幻世界，中世纪，四元素魔法体系。"
story_start: "深夜，林远在家族废墟发现了父亲留下的密信…"
story_end: "林远找到炎之心，击败幕后黑手，成为火焰守护者。"

# 字数与版本
target_word_count: 8000
num_versions: 3

# 约束
max_chapters: 30
max_npcs: 30
total_token_budget: 0

# 主角团（可选）
protagonist_mode: "spotlight"
protagonist_count: 1
protagonist_order: ""

# NPC（可选）
npc_mode: "parallel"
npc_name_style: "chinese"
user_npcs:
  - name: "铁山"
    role: "friend"
    personality: "沉默忠诚的佣兵"
    relevance: "high"

# 命名与风格（可选）
naming_style: ""
writing_style: ""
quality_mode: "balanced"
front_matter: ["toc"]

# API
model: "claude-sonnet-4-6"
api_provider: "anthropic"
```
